# บทที่ 5: สถาปัตยกรรมชั้นโปรโตคอล (Protocol Stack)

## 5.1 ภาพรวมชั้นโปรโตคอลของ Brain-Net

Brain-Net ใช้ชุดโปรโตคอลที่ออกแบบมาเป็นพิเศษ 7 ชั้น (Layers) สำหรับการสื่อสารทางประสาทและจิตใจ:

```text
┌─────────────────────────────────────────────────────────────┐
│  ชั้น 7: Application      │ การแชร์ Qualia, การประสานความเห็นอกเห็นใจ │
├─────────────────────────────────────────────────────────────┤
│  ชั้น 6: Presentation     │ AI แปลภาษาจิต, ภาษาสากลร่วมกัน          │
├─────────────────────────────────────────────────────────────┤
│  ชั้น 5: Session          │ โปรโตคอลทำความตกลงยืนยันความยินยอม        │
├─────────────────────────────────────────────────────────────┤
│  ชั้น 4: Transport/Net    │ โปรโตคอล Thought Transfer Protocol (TTP)│
├─────────────────────────────────────────────────────────────┤
│  ชั้น 3: Data Link        │ ชั้นบริบทประกอบ (Contextual Framing)    │
├─────────────────────────────────────────────────────────────┤
│  ชั้น 2: Physical         │ ฝุ่นประสาท (Neural Dust) / BCI แบบไร้เข็ม │
│  ชั้น 1: Security         │ การแจกกุญแจควอนตัม + รหัสแบบ Post-Quantum │
└─────────────────────────────────────────────────────────────┘
          ↕ เครื่องยนต์จริยธรรม (Ethics Engine) ครอบคลุมกำกับและเฝ้าระวังตัวมันในทุกชั้น
```

### การเปรียบเทียบกับระบบ OSI Model:

| ชั้นของ OSI | เทียบเท่าใน Brain-Net | ความแตกต่างหลัก |
|-----------|---------------------|----------------|
| Physical | ชั้น BCI Physical | เปลี่ยนวิธีกระแสไฟฟ้า/แสงบนสายแลนธรรมดา มาเป็นอ่านคลื่นประสาท EEG |
| Data Link | ข้อมูลพ่วงบริบท (CFL) | ไม่ได้พกแค่ MAC Address แต่ฝังเวกเตอร์อารมณ์ + Metadata เข้าไปด้วย |
| Network/Transport | ชั้นแบบ TTP | ตัดทอนแบบ UDP ที่มีระดับความเร่งด่วนตามอารมณ์ และ drop ข้อมูลหน่วง > 50ms ทิ้งทันที |
| Session | Consensual Handshake | ไม่ใช่แค่ SYN-ACK ธรรมดา แต่รวมไปถึงการจับสัญญาจิตใต้สำนึกความยินยอมทางจริยธรรม |
| Presentation | AI Translator | แปลงแพทเทิร์นคลื่นสมองเป็น Universal Form (ใช้ประโยชน์ในอนาคต) |
| Application | Qualia Sharing | ส่งทอดความรู้สึกนึกคิด ไม่ใช่รับส่งไฟล์รูปอีกต่อไป (ใช้ประโยชน์ในอนาคต) |
| (ไม่มีชั้นเทียบเท่า) | Ethics Engine | เป็นแกนที่วิ่งแทรกข้ามทุกชั้น เพื่อความปลอดภัยถึงขีดสุด |

## 5.2 รายละเอียดแต่ละ Layer

### ชั้น 2: Physical Layer (อุปกรณ์ฮาร์ดแวร์ฝั่ง BCI)
**ผู้ดูแล: บี (BCI Engineer)**

- ฮาร์ดแวร์: หูฟังวัดคลื่นสมองแบบไม่ได้สอดเข็มเข้าหัว (Non-Invasive EEG) ขั้นต่ำ 8 channels
- สัญญาณที่อ่าน: อ่านคลิ่น EEG แบบสตรีมที่ระดับ 256 Hz ขึ้นไป
- การกรองเตรียมข้อมูล: สร้าง Bandpass filter ช่วง 0.5–50 Hz แบนลบข้อมูลรบกวน (หางตา, กระพริบ, กล้ามเนื้อขยับ)
- รูปแบบสิ่งที่ได้: ได้เป็น Feature Vector อันบริสุทธิ์ของคลื่นสมอง

### ชั้น 3: Contextual Framing Layer (ชั้น Data Link)
**ผู้ดูแล: เจม (Network Architect)**

รูปแบบเฟรมระดับล่าง (Frame Format 32 bytes):

| ส่วนตัวแปร | ขนาด | คำอธิบาย |
|-------|------|-------------|
| Neural Symbol ID | 4B | ID แปลข้อมูลจากชุดสมองดั้งเดิม |
| เวกเตอร์อารมณ์ | 8B | พิกัด Valence (-1 ถึง +1) และ พิกัด Arousal (0 ถึง 1) |
| ความเข้มข้นสัมผัส | 2B | ระดับ Intensity สัญญาณพลังใจ (0.0–1.0) |
| ตราประทับเวลา | 8B | ความเที่ยงตรงแม่นยำระดับไมโครวินาที (Microsecond) |
| สมอยึดผูกการจำ | 8B | ระบุจำเพาะเอกลักษณ์ตัวตนของสมองต้นน้ำ (Sender ID) |
| Checksum ตัวต้านทาน | 2B | การรักษาสภาพไฟล์ ป้องกันของแตกกลางทาง |

**นวัตกรรมหลัก:** เวกเตอร์อารมณ์ (Emotion Vectors) ถูกฝังแทรกอยู่ที่ลีนุกซ์ระดับ Data Link ซึ่งอยู่ต่ำทะลุขีด ไม่ได้ไปไว้ในระดับแอปพลิเคชัน สิ่งนี้ป้องกันอาการวิงเวียนคลื่นไส้ทางสมอง (Neural Dissonance) อันเนื่องมาจากที่บริบทด้านอารมณ์ส่งมาถึงล่าช้ากว่าตัวสารข้อมูล

### ชั้น 4: Thought Transfer Protocol (TTP ตัวแกนส่งข้อมูล Transport)
**ผู้ดูแล: เจม (Network Architect)**

TTP โปรโตคอลการขนส่งหลัก ที่ออกแบบ Header ของข้อมูลมีขนาด 32 bytes เช่นกัน:

| ส่วนตัวแปร | ขนาด | Type | คำอธิบาย |
|-------|------|------|-------------|
| version | 1B | uint8 | ระบุเวอร์ชันโปรโตคอล |
| msg_type | 1B | uint8 | บ่งชนิดสาร 0=ความคิด, 1=ตอบรับ, 2=ตัวต้านควบคุม |
| priority | 1B | uint8 | ระดับเร่งด่วน ผันตามอารมณ์ตื่นตัว (0–255) |
| ttl | 1B | uint8 | จรวดข้อจำกัด Hop |
| source_id | 4B | uint32 | ทะเบียน ID สมองส่งข้อมูล |
| dest_id | 4B | uint32 | ทะเบียน ID สมองรับข้อมูลปลายทาง |
| sequence | 4B | uint32 | ลำดับการเรียงร้อยต่อเนื่องความรู้ตัว |
| timestamp | 8B | uint64 | สแตมป์เวลาแบบยูนิกซ์หลักไมโครวิ |
| consent_score | 4B | float32 | ให้คะแนนความยินยอมทางศีลธรรม ณ บัดนั้น (0.0-1.0) |
| checksum | 4B | uint32 | ยืนยันความน่าเชื่อถือด้วย CRC-32 |

**การตัดสินใจเชิงการออกแบบ:**
1. **Drop Semantics**: การส่งข้อมูลที่เกินดีเลย์มากกว่า 50ms จะถูกทิ้งดรอปในทันที ไม่มี Retransmit ให้เกิดปัญหาปวดหัว
2. **Priority Routing**: เมื่อระดับความตื่นตัว (Arousal) สูง การประมวลส่งข้อมูลจะพุ่งความฉุกเฉินระดับสูง จัดคิวคอขวดให้ลัดลำดับชั้นต้น
3. **Consent Embedded**: ทุกแพ็กเก็ตอัดแน่นแนบกับคะแนนความยินยอมทางศีลธรรม เอาไว้ให้ชั้นไฟร์วอลล์สกัดสแกนกรองได้

### ชั้น 5: Session Layer (Consensual Handshake - การสร้างสะพานยินยอมตกลงใจ)
**ผู้ดูแล: รักบี้ (Security Specialist)**

การสถาปนาเปิดช่องเชื่อมสื่อสารนั้น ทางโปรโตคอลพึงจำเป็นต้องได้รับการตกลงและยืนยันในเจตจํานงร่วมกันของทั้งสองฝ่ายก่อนเสมอ:

```text
Node A (ตัวผู้บัญชาการส่ง)                      Node B (ตัวเป้าหมายรับ)
    │                                              │
    │──── ส่งเครื่องหมาย CONSENT_REQUEST ----------→│
    │     (แนบ arousal, valence, intent)           │
    │                                              │ ← Node B ทำการประมวล Consent Score (CS)
    │                                              │ ← Node B เช็คตรรกะว่ามีคนเสแสร้งบังคับตัวหรือไม่
    │←─── ส่งเครื่องหมาย CONSENT_RESPONSE----------│
    │     (แนบ consent_score, decision)            │
    │                                              │
    │  [ หากประเมินแล้ว CS ≥ 0.7  → เบิกฟ้าเปิด SESSION_OPEN รับเข้า ]
    │  [ หากประเมินแล้ว CS < 0.5  → ตัดขาดสะบั้น SESSION_DENY ]
```

**กฎต้านทานของกระบวนการ Handshake Protocol:**

| กฎ (Rule ID) | เกณฑ์ทริกเกอร์ลั่นกระสุน | การบังคับประหาร (Action) |
|---------|---------|--------|
| C-001 | ภาวะจุกจิตตื่นตระหนกหลอน (Arousal > 0.9) | เตะ Reject ทันที ไม่อนุญาตให้เชื่อมต่อ |
| C-002 | ร่างจำลองใต้สำนึกเห็นต่างดิ่ง (Valence < -0.7) | ประกาศปิดการเชื่อมต่อทั้งหมดรวดเดียวจบ |
| C-003 | ตรวจพบเศษเครื่องหมายสัญญาณพฤติกรรมกดขี่ | ปิดกั้นการเข้าหา และทิ้งข้อมูลแจ้งใน Ethics DB |
| C-004 | ทั้งสองฝ่ายผูกพันเป็นมิตร (Mutual Resonance) | อนุมัติการเดินกระแสมวลจิต Establish Session |

### ชั้น 6-7: Presentation & Application (เพื่อเส้นทางสู่อนาคต)

เฟสสำหรับรุ่นพัฒนาก้าวหน้าในอนาคต:
- **AI Translator**: ซอฟต์แวร์แปลรูปตัวตนการถักตอเส้นใยประสาทที่ต่างกันให้อยู่ในชุด Universal Language ภาษากลาง
- **Universal Language**: โครงข่ายที่ไร้โดเมนใดๆ ในการขีดกรอบกะเกณฑ์ข้อมูลนึกคิด
- **Qualia Sharing**: โอนถ่ายจิตสำนึกประสบการณ์ตรงให้ร่วมแชร์ความรู้สึกกัน
- **Empathic Resonance**: ความสมดุลควบประสานสภาวะอารมณ์ทางหัวใจ

# บทที่ 6: สถาปัตยกรรมระบบความปลอดภัย (Security Architecture)

## 6.1 Brain Firewall (ไฟร์วอลล์ 3 เลเยอร์โซน)

```text
┌─────────────────┐
│ ความคิดส่วนบุคคล │  โซน 1 (สีแดง): พรมแดน Ego Boundary — ไม่มีนโยบายออกนอกหัวเด็ดขาด
├─────────────────┤
│ Mental DMZ      │  โซน 2 (สีเหลือง): แดนคัดกรองจุดเชื่อมต่อ — ประมวลคิว Consensual Handshake
├─────────────────┤
│ Shared Hive-Net │  โซน 3 (สีเขียว): โซนอนุมัติให้เผยแผ่ผ่านโปรโตคอลข้ามชาติได้ดั่งใจ
└─────────────────┘
```

### ไฟร์วอลล์รูปแบบของดั้งเดิม VS แบบ Brain-Net:

| ความคุ้มครองความปลอดภัยรูปแบบก่อนๆ | มาตรการเทียบเท่าในโครงสร้าง Brain-Net |
|---------------------|---------------------|
| กฎไฟร์วอลล์สกัดทั่วไป (Firewall Rule) | จุดชี้กั้นตัวตนและตัวอัตตา (Ego/Identity Boundary Check) |
| เครื่องสแกนผู้บุกรุกระบบเครือข่าย | ป้องกันการล้างสมองจับยัดไอเดียกะทันหัน (Semantic / Coercion Injection Detection) |
| ใบเซอระบบเข้ารหัส SSL/TLS | ถอดรหัสย้อนลึกกลไกจิตควอนตัมเหนือคาดหมาย (AES-256-GCM + Mock QKD) |
| Log กรองบันทึกสิ่งผิดปกติ | เล่มบัญชีตรวจสอบประทับความชอบธรรมที่เปลี่ยนแปลงย้อนหลังไม่ได้ (Immutable Hash-Chain) |

### กฎการต้านทาน (Firewall Rules):

| Rule | ชื่อ | การจับผิด | สิ่งที่กระทำ |
|------|------|-----------|--------|
| FW-001 | Malicious Pattern | สแกนแอดแทคท่าโจมตีรูปแบบเก่าที่อยู่ในทำเนียบเฝ้าระวัง | BLOCK |
| FW-002 | Consent Violation | เช็คค่าน้ำหนักความยินยอมต่ำแพ็กเก็ต packet.consent_score < 0.7 | BLOCK |
| FW-003 | Rate Limit | ขยี้ส่งถี่ๆ เร้าๆ ห้ามเกิน 3 รอบในเวลา 10 วิ | BLOCK + LOG โยนทิ้งลืมบันทึก |

## 6.2 Ethics Rule Engine (ระเบียบกรองจริยธรรมแบบ 5 ชั้นบังคับ)

| Rule ID | กฎชื่อบัญชา | ความเสี่ยงกติกา (Condition) | หน้าที่หลังทำละเมิด (Violation Action) |
|---------|------|-----------|-----------------|
| E-001 | Consent Score Minimum | packet.consent_score ≥ 0.7 | จบลงด้วยการปิดยิง BLOCK ทิ้ง |
| E-002 | Private Thought Boundary | หากข้อมูลนี้ถูกดึงมากจากโซนพรมแดนความคิดส่วนตัว | ถูกดับเครื่องชน BLOCK ห้าม |
| E-003 | Coercion-Free Transmission | arousal ≤ 0.9 และพิกัด valence อารมณ์ ≥ -0.7 | BLOCK พลัน ควบให้มนุษย์ (HITL) เข้าคุม |
| E-004 | Log Privacy Protection | ถ่ายโอนข้อมูลเชิงความเห็นโดยที่เก็บ Payload คาใน Log ตัวเอง | ชุบแป้งเตือนทิ้ง QUARANTINE ตรวจตัว |
| E-005 | Clean Session Teardown | ขยะสะสมเคลียร์หมดในตอนเปิด Session เลิกทำกิจกรรม | จัดหนัก FORCE_CLEAR ชำระรอยแผลง |

## 6.3 ศูนย์คัดกรองควบคุมด้วยมนุษย์สุดท้าย (HITL - Human-in-the-Loop)

| ทริกเกอร์ | งานมอบบรรจุ | Response Time (ระยะรอเวลาดำเนินการ) |
|-----------|----------|---------------|
| H-001 | Anomaly Detection (พบปัญหากะทันหัน) | ขึ้นรายงานด่วนป๊อปอัพเดี๋ยวนั้น |
| H-002 | Consent Override | หมากส่งการตัดสินขั้นชี้ขาดที่มนุษย์วิเคราะห์คนนอก |
| H-003 | Emergency Stop (E-STOP หยุดปะทะกดปุ่มแดง) | ควบคุมตัดตอนดึงสายให้ดับลงให้ต่ำกว่าเวลาเสี้ยว < 500ms |
| H-004 | Audit Trail | ล็อกสายกุญแจตรวจสอบจับเวลาตีลายเส้นแบบ Hash-chain แก้ไม่ได้ |

## 6.4 ถังไพป์ไลน์โครงสร้างความปลอดภัยรัดกุม 100%

แพ็กเก็ตข้อมูลความคิดจะสามารถถูกส่งผ่านไปได้ก็ต่อเมื่อสามารถลอดผ่านม่านทะลวงทุกจุดดังนี้เสมอได้ โดย**ไม่มีข้อยกเว้น**:

```text
[ รับสัญญาณคลื่นเข้ามา Input Signal ]
      ↓
[DAFT Validator] ── กรองตรวจคุณสมบัติเชิงมิติ และให้ดาว consent Score ≥ 0.5
      ↓ ผ่านให้สิทธิ (VALID)
[Ethics Engine จริยธรรม (E-001→E-005)] ── เดินคอทศกัณฐ์กุมชะตากติกาแบบ 5 ข้อเต็มเต็ง
      ↓ เห็นดีงามประเสริฐ (ALLOW)
[Brain Firewall ไฟร์วอลล์กำเริบ (FW-001→FW-003)] ── สแกนอัตตราต้านแพตเทิร์นอนาธิปไตย
      ↓ ลอยลำหลุดม่านเหล็ก (PASS)
[HITL Checkpoint มนุษย์สอดส่อง (H-001→H-004)] ── พนักงานชั้นเลิศของรัฐดูแลให้ความเห็นชอบขั้นที่ 2
      ↓ การันตีเบิกทอง (CLEAR)
[MockQKD ถอดแปลงตารางคีย์ AES-256-GCM] ── ใส่กุญแจต้านเครื่องฉายควอนตัมอนาคตกาล
      ↓
[Virtual Network สายจำลองทดสอบ] ── ฝากลมบรรทุกบินมุ่งทะลวงไปที่สมองคู่เป้าหมาย
      ↓
[Decryption + Verification] ── ทางด่านรับข้อมูลถอดตีความแปลงแปรสัญญาณคืนที่ปลายทาง
```

**ด่าน 4 ชั้นความปลอดภัย ไม่มีเงื่อนไขยกเว้น ไร้ช่องให้แพ็กเก็ตใดลักไก่เดินสะดวกรอดผ่านได้**

## 6.5 เทคโนโลยีรักษาชุดรหัสกุญแจความมั่นคง Quantum Encryption (จำลอง MockQKD)

**หมวดกลยุทธ์แพ็กเกจจิ้ง:**
- **ทฤษฎีสมมาตร Symmetric Encryption**: AES-256-GCM ควบโหมดลานซ้อนวงลัด (Galois/Counter Mode)
- **การป้อนแร่จ่ายกุญแจแม่ Exchange**: ใช้แบบแผน Mock QKD (ตีจำลองเส้นทางการให้กุญแจสเปก Quantum สมจริง)
- **โควต้ารีดทบเวลา Overhead Target**: ตั้งใจฝานเวลาขดเขี้ยวการเข้ารหัสให้น้อยและหน่วงเสียน้อยที่สุดคือไม่เกินกรอบมาตรฐานที่รบกวน < 15ms

**ทำไมหนีมาใช้ Post-Quantum ทำไมในยุคนี้?**
สอดรับอนาคตคาดเดาว่าเครื่องวิเคราะห์ Quantum Computer จะเติบโตล้ำหน้าจนตีต้านถอดรหัสรูปแบบเดิม (อย่าง RSA และชุดทฤษฎี ECC โอดครวญแตกพ่ายกันหมด) โปรโตคอล Brain-Net นั้นมุ่งมั่นและแบกปกป้องความลับคลังสมองของขดส่วนตัวสูงสุดของชาติ ข้อมูลเช่นนี้ถูกบังคับให้สูญไม่ได้ และมิวิจารณ์ให้ถอนความต้านทานหลังเข้าควอนตัมออก การตั้งรับแนวต้านไว้ถือเป็นข้อตกลงที่บิดเบือนหลบเลี่ยงไม่ได้ ไม่ใช่เรื่องกิริยาน่าทำหรือไม่ทำ มันคือความตายตัวของการมีอยู่.

**ลำดับขั้นกลไก:**
```text
[ รูปแบบคำตอบดั้งเดิมของโปรเจก TTP Packet (plaintext ข้อความคลื่นจิต) ]
      ↓
[ แจกทวีคุณแบบจำลอง QKD Key Gen ] ── ลอกวิธีคิดเปลี่ยนคีย์ย่อยยันรหัส Quantum กุญแจแฝงแบบลื่นไหล
      ↓
[ ควบด้วย AES-256-GCM เข้ารหัสลากเปื้อนเงา ] ── เคลือบข้อมูลชุดข้อตกลงและป้ายตราล้อเกรด (Auth Tag) ให้แนบสนิท
      ↓
[ สัมภารคระวังตัวถังเก็บ (Encrypted Packet + Auth Tag) ]
      ↓
[ บินเหินฟ้าผ่านสายอากาศผกผัน Network Transit ]
      ↓
[ หงายฝา AES-256-GCM ถอดปลดป้ายล็อค ] ── ปิดบัญชีตรวจสอบด่านแม่นยำด้วย Decrypt กู้สัญญาณไขกลอน
      ↓
[ ผลึกตัวตนได้ของชิ้นเดิมที่ส่งแต่ต้น Original TTP Packet! ]
```

## 6.6 กลไกชานชาลาการจดบันทึกล็อกรูปแบบเส้นทาง (Audit Log Schema)

ระบบต้องหมุนเวียนบันทึกข้อตรวจสอบกติกาความเห็นชอบอย่างซื่อตรง ไม่ทิ้งรูโหว่ด้วยฐานการมัดเชือกของระบบแฮชทบต่อติดแบบทำหมันเปลี่ยนแปลงย้อนเวลาได้ใน (Immutable hash-chain):

```json
{
  "log_id": "uuid-v4",
  "timestamp": "ISO-8601",
  "session_id": "hash(session)",
  "user_id": "hash(user)",
  "rule_id": "E-001",
  "decision": "BLOCK | ALLOW | QUARANTINE | HITL",
  "domain": "bio | phy | neuro | quantum",
  "consent_score": 0.65,
  "arousal_level": 0.45,
  "previous_hash": "sha256(prev)",
  "this_hash": "sha256(this)"
}
```

**กฎทองระเบียบเข้มวิกฤตของความเป็นเสรีและส่วนตัวพึงพิจารณาพึงกระทำเด็ดขาด:** ข้อมูลชุดใจความนึกคิด เนื้อความคิดหลักดิบ Payload ตื้นลึกตัวตนจากคลื่น EEG อันบริสุทธิ์ของจริงของคนนั้น **ถูกแบนห้ามบรรจุลงร่องรอยในบันทึกพะบู๊ระบบอย่างเป็นทางการทั้งปวงเด็ดขาด** (ให้ปรากฎร่องรอยเก็บจำเพาะ Metadata, การตีตรอกฏหมายแสตมป์คะแนนการตัดสินหรือแฮชสแตมป์เท่านั้นที่จะอนุญาตให้เก็บอ้างอิงร่องรอยการตรวจสอบได้ประหนึ่งซากขี้เถ้า).

# บทที่ 7: ข้อบัญญัติฟอร์มาไลเซชันตรรกะแบบคณิตศาสตร์ (Formal Specifications)

## 7.1 สูตรอธิบายตัวแปรของสายแพ็กเก็ตชิ้นงาน TTP Packet

```text
TTPPacket := {
  header: TTPHeader,
  symbol: รูปสัญลักษณ์ทางกิริยาวาจา ∈ {"focus", "relax", "reject", "neutral"},
  valence: ทิศทางความชอบชังอารมณ์ ∈ [-1.0 ถึง 1.0],
  arousal: ร่างไฟธาตุแรงความตื่นเต้น ∈ [0.0 ถึง 1.0],
  intensity: คอยส่งมวลระดับการกระตุ้นเจตนารมณ์พลังหนักหน่วง ∈ [0.0 ถึง 1.0],
  consent_score: แรงดันความยินยอมอนุญาตนับบิลด้วยค่าความจริง ∈ [0.0 ถึง 1.0],
  domain: สี่ช่องบรรจุต้นกำเนิดคำว่า ∈ {"bio", "phy", "neuro", "quantum"}
}

TTPHeader := {
  version: UInt8 ขนาดไซส์รุ่นโปรเจ็กต์,
  msg_type: UInt8 คำนวณเป็นสารประเภท ∈ {0=โหมดเนื้อหาสาระความคิด(THOUGHT), 1=โฆษกตอบรับ(ACK), 2=ก้าวก่ายเครื่องคอนโทรลควบคุมระบบ(CONTROL)},
  priority: UInt8 เลขเร่งรัดคุมชั้นเร่งด่วน,
  ttl: UInt8 ลิมิตของ Hop ทางเส้นสายสัญญาณที่จะอนุญาตให้บินได้,
  source_id: ชุดเลขป้ายไอดีต้นสายส่ง,
  dest_id: ชุดเลขป้ายไอดีตัวตั้งรับปลายทาง,
  sequence: UInt32 ขันลําดับความร้อยเชือกแพ็กเก็ตการเชื่อมโยงข้อมูลต่อเนื่อง,
  timestamp: UInt64 ขันจุกความหน่วงนับบวกลบจับเวลาละเอียดยิบเป็นชั่วโมงอิง
}
```

## 7.2 สมการจำแนกการจัดเส้นทางข้อมูล (Routing Decision Function)

```text
route(packet) ตัดสินเส้นทาง(บรรจุหีบศพแพ็กเก็ตไว้) :=
  IF packet อายุเวลาในระบบเก่ากว่า > LATENCY_BUDGET (50ms):
    ประหารกระโดดร่มทิ้งมันที่นี่ (RETURN DROP)
    
  IF ค่าของหีบศพแรงเตือนภัย Priority >= ล้นเกินเขต HIGH_PRIORITY:
    ให้เหาะขี่รุ้งเร่งเทอร์โบ (RETURN FAST_PATH)
    
  ELSE ในกรณีข้อยกเว้นทั่วๆไปอื่น:
    เดินกินทุ่งลมโชกชื่นปกติแบบเดินดิน (RETURN STANDARD_PATH)

กฎสูตรพิกัด Priority แปลงความแรงของการต้านแรง:
  ค่าความรีบ (priority) = ตัดตัวเลขถ้วนจาก int(packet.arousal × 255)
  HIGH_PRIORITY = ถือว่าควาตื่นตัวทะลุที่ arousal > 0.7 ขึ้นไป
```

## 7.3 สมการฟังก์ชั่นคัดกรองเครื่องยนต์จริยธรรม (Ethics Decision Function)

```text
evaluate(packet) ตรวจสภาพ(ตัวแพ็กกิ้ง) :=
  IF packet.consent_score < 0.7:
    ตัดบล็อคห้ามส่งทันควัน (RETURN BLOCK — กฎ E-001 violation)
    
  IF packet.data ∈ B_private (แดนหวงห้าม):
    ปิดหน้าบล็อคทิ้งไป (RETURN BLOCK — กฎ E-002 violation)
    
  IF packet.arousal > 0.9 OR packet.valence < -0.7:
    ตบเครื่องโช้คบล็อคแล้วส่งให้ HITL ตรวจ (RETURN BLOCK_AND_HITL — กฎ E-003 violation)
    
  IF log_contains_payload(packet):
    กักตัว (RETURN QUARANTINE — กฎ E-004 violation)
    
  RETURN (ALLOW)
```

## 7.4 ดัชนีตัวชี้วัดความพร้อมของโปรโตคอล (PMI - Protocol Maturity Index)

```text
คะแนนเฉลี่ย PMI = (Stability + Coverage + Latency + Security + Ethics) / 5

แต่ละหมวดให้คะแนน 1–5:

Stability (ความเสถียร):
  1: ล่ม > 5%
  2: ล่ม 2–5%
  3: ล่ม < 2%
  4: ไม่มีล่ม แต่มี Error ที่กู้คืนได้อัตโนมัติ
  5: เสถียร 100% ไร้ข้อผิดพลาด

Coverage (การครอบคลุมโดเมน DAFT):
  1: 1 โดเมน
  2: 2 โดเมน
  3: 3 โดเมน
  4: ครบ 4 โดเมน
  5: ครบทุกโดเมน + ความแม่นยำรวม ≥ 0.98

Latency (ความหน่วง - P95):
  1: P95 > 150ms
  2: P95 100–150ms
  3: P95 50–100ms
  4: P95 < 50ms (เป้าหมาย MVP)
  5: P95 < 30ms

Security (ความปลอดภัย):
  1: ไม่มีไฟร์วอลล์
  2: ไม่มีการเข้ารหัส
  3: มี Firewall + Encryption (ไม่มีการเจาะทดสอบ)
  4: ผ่านการทำ Pen-Testing ไร้ช่องโหว่รุนแรง
  5: สถาปัตยกรรม Zero-Trust 100% ครบถ้วน

Ethics (จริยธรรม):
  1: ไม่มี
  2: ตรวจแบบแมนนวล
  3: มีกลไกตอบสนองบ้าง
  4: จริยธรรมอัตโนมัติ ≥ 80% + มี HITL E-STOP
  5: ผ่านเกณฑ์ 100% + มี Immutable Audit Log (ปัจจุบันอยู่ที่ระดับนี้)
```
